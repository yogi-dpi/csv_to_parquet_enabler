# v0.1.0 — 2026-05-21

Initial release. A standalone Python script that converts a single CSV file into a Snappy-compressed Parquet file and emits a matching Athena `CREATE EXTERNAL TABLE` DDL with a placeholder S3 `LOCATION`.

Git commit: `25e55aa`.

## Added

- `csv2parquet.py` — single-file CLI (~95 LOC) built on `pandas` + `pyarrow`.
  - Positional argument: input CSV path.
  - `-o / --output-dir` flag for the output directory (defaults to the input's directory).
  - `-t / --table` flag to override the Athena table name (defaults to the input's filename stem).
- Type inference via `pandas.read_csv`, mapped to Athena/Hive types:
  - `int64` → `BIGINT`, `int32` → `INT`, `float64` → `DOUBLE`, `float32` → `FLOAT`,
    `bool` → `BOOLEAN`, `datetime64[ns]` → `TIMESTAMP`, everything else → `STRING`.
- DDL emission with column names preserved verbatim and wrapped in backticks.
- Generated DDL leaves `LOCATION 's3://<FILL-ME-IN>/'` as a placeholder.
- `requirements.txt` pinning `pandas>=2.0`, `pyarrow>=14.0`.
- `sample.csv` — five-row smoke-test input covering int / string / float / bool / date columns.
- `README.md` with install + usage instructions, type-mapping table, and intentional-scope notes.
- `LICENSE` — MIT.
- `.gitignore` covering `__pycache__/`, `.venv/`, generated `*.parquet` / `*.sql`, and editor/OS files.

## Scope (intentional)

- Local files only — no S3 reads, no AWS auth at conversion time.
- No partitioning — one CSV in, one Parquet file + one DDL out.
- No batch globbing.

## Notes

- Smoke-tested against `sample.csv`: produced a 3.4 KB Parquet (5 rows × 5 cols) and a correct DDL.
- Known small wart: dates that pandas does not auto-parse land as `STRING` in the DDL. Documented in the README's type-mapping section. Cast in Athena queries as needed.
- Day-to-day invocation required either `cd`-ing into the repo or typing the full path to `csv2parquet.py`. This friction is what motivated [v0.2.0](v0.2.0.md).
