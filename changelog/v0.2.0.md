# v0.2.0 — 2026-05-21

Make the tool installable as a real CLI: after `pip install -e .`, the `csv2parquet` command is on your PATH and runs from any directory — no more typing the script's path.

Git commits: `48ba8ca`, `96adfb1`.

## Added

- `pyproject.toml` declaring:
  - Project metadata (`csv-to-parquet-enabler`, version, license, Python `>=3.9`).
  - Runtime dependencies `pandas>=2.0`, `pyarrow>=14.0` (same as `requirements.txt`).
  - **`[project.scripts]` entry** `csv2parquet = "csv2parquet:main"` — the line that creates the `csv2parquet` executable shim on install.
  - `py-modules = ["csv2parquet"]` so setuptools picks up the single-file module at the repo root.
- "Run from anywhere" section in the README explaining the installable flow.
- Fallback section in the README for users who prefer running the script directly without installing.

## Changed

- README usage examples now use `csv2parquet sales.csv` instead of `python csv2parquet.py sales.csv`.
- Install instructions now say `pip install -e .` instead of `pip install -r requirements.txt`. `requirements.txt` is retained for the no-install fallback path.
- `.gitignore` now ignores editable-install / build artifacts: `*.egg-info/`, `build/`, `dist/`.

## Unchanged (worth calling out)

- `csv2parquet.py` itself — **zero changes to logic**. `main()` was already the entry point; v0.2.0 just told the packaging system about it.
- Type mapping, DDL template, output file naming, Snappy compression — identical to v0.1.0. v0.2.0 produces byte-identical Parquet/DDL output for any given CSV.
- Project scope — still local-only, still no partitioning, still preserves column names verbatim.

## Migration from v0.1.0

```bash
cd csv_to_parquet_enabler
git pull
source .venv/bin/activate
pip install -e .          # one-time
# from now on:
csv2parquet some.csv      # works from anywhere
```

No data migration needed — generated files are bit-for-bit compatible with v0.1.0.

## Notes

- Rationale: v0.1.0 added a small friction (remember the script's path) to every single invocation. Across a workflow where the user converts CSVs multiple times, that friction compounds. v0.2.0 trades five minutes of one-time setup for zero friction at every subsequent use, on every machine.
- Verified post-install by running `csv2parquet demo.csv` from `/tmp` after copying `sample.csv` → `/tmp/demo.csv`; outputs landed in `/tmp/` as expected.
